# 回合选择与剪辑段算法

## 选择

Main 从最近一次经验证的 `AnalysisResultV1` 重新选择回合，Renderer 不能提供任意时间段。

- **所有回合**：全部有效回合。
- **精彩回合**：严格使用 `bounce_count > X`，X 只允许 3、5、7。
- **自定义**：只接受结果中存在的回合 ID；空选择拒绝执行。

选择前按 ID 精确去重，再按开始时间和稳定 index 排序。当前 `bounce_count` 是弹跳代理，不是实际击球计数。

## 原始 5 秒分组

对相邻回合计算：

```text
gap = next.start_time_seconds - currentGroup.rawEnd
```

- `gap < 5.0`：合并到当前原始组；
- `gap >= 5.0`：开始新组；
- 恰好 5.000 秒不合并。

比较使用极小浮点容差，确保 4.999 合并而 5.000 分开。

## 前后扩展与二次去重

每个原始组只扩展一次：

```text
start = max(0, rawStart - preRoll)
end   = min(videoDuration, rawEnd + 1.0s finalRallyTail + postRoll)
```

每个剪辑片段只在组内最后一个回合结束后增加 1 秒固定收尾，再完整应用用户选择的回合后时间；组内较早回合不增加这 1 秒。前置时间仍只允许 1.5、2.5、5 秒，后置仍只允许 0.5、1、2、4 秒。若所需结束时间超过源视频末尾，则直接在源视频末尾结束。若扩展后的下一组 `start <= previous.end`，再次合并，以避免重复画面。

非法/非有限时间、结束不晚于开始、重复 ID、空结果和无效视频时长均不会启动 FFmpeg。

## 已覆盖边界

- 4.999、5.000 和大于 5 秒；
- 三个连续短间隔；
- 乱序和重复回合；
- 视频首尾截断；
- 扩展后重叠；
- 严格精彩阈值；
- 自定义空选择。
